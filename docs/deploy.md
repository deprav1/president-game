# Деплой и публикация в Telegram

Игра «Варония» — **полностью статическое** клиентское приложение (React + Vite,
без бэкенда, без БД, без API). Telegram Mini App просто открывает HTTPS-страницу
во встроенном webview. Поэтому **сервер не нужен** — хватает бесплатного
статик-хостинга на CDN.

## Нагрузка

100–200 пользователей в день — это микронагрузка для CDN (статика держит сотни
тысяч заходов). Единственный ресурс, который тратится, — **трафик на картинки**.
После перевода в WebP сборка весит ~3.9 МБ, из них ~3.3 МБ — изображения,
кэшируемые браузером (см. `public/_headers`). При 200 заходах/день первичный
трафик ≈ 0.7 ГБ/день ≈ 20 ГБ/мес — в пределах любого бесплатного тарифа.

## Где хостить (бесплатно)

| Хостинг | Трафик/мес | Особенности |
|---------|-----------|-------------|
| **Cloudflare Pages** ⭐ | безлимит | Быстрый CDN в РФ/СНГ, мгновенный HTTPS. Рекомендуется как основной. |
| GitHub Pages (текущий) | 100 ГБ | Уже настроен (`.github/workflows/deploy.yml`). Держать запасным. |
| Vercel / Netlify | 100 ГБ | Альтернативы, тоже из коробки. |

### Cloudflare Pages — настройка (один раз)

1. dash.cloudflare.com → **Workers & Pages** → **Create** → **Pages** →
   **Connect to Git** → выбрать репозиторий `president-game`.
2. Параметры сборки:
   - **Framework preset:** Vite
   - **Build command:** `npm run build`
   - **Build output directory:** `dist`
   - **Production branch:** `main`
3. Deploy. Через ~1 мин будет URL вида `https://president-game.pages.dev`.
   Это и есть HTTPS-URL для Telegram.

Файлы `public/_headers` (кэш) и `public/_redirects` (SPA-fallback) Cloudflare
подхватывает автоматически. GitHub Pages их игнорирует — это безопасно.

## Подключение к Telegram (чек-лист)

1. **@BotFather** → `/newbot` (если бота ещё нет) → получить токен.
2. **@BotFather** → `/newapp` → выбрать бота → задать название, описание,
   иконку (640×360) и **Web App URL** = ваш hosting-URL (например
   `https://president-game.pages.dev`).
3. Получите ссылку вида `https://t.me/ВашБот/ИмяApp` — это прямой запуск игры.
4. (Опционально) **@BotFather** → `/setmenubutton` → привязать кнопку меню
   бота к тому же URL, чтобы игра открывалась из чата с ботом.
5. Проверка в клиенте Telegram (моб. + десктоп):
   - игра открывается на весь экран (вызывается `WebApp.expand()`),
   - тема и цвета подхватываются,
   - тактильная отдача (haptics) работает на телефоне,
   - реферальные ссылки `?startapp=...` разворачиваются (`initDataUnsafe.start_param`).

## Что уже есть в коде (готовность к Telegram)

`src/main.jsx` и `src/App.jsx` уже используют Telegram WebApp SDK:
`ready()`, `expand()`, `requestFullscreen()` (для клиентов ≥ 8.0),
`HapticFeedback`, `openLink()` и чтение `start_param` для рефералов.
Скрипт `telegram-web-app.js` подключён в `index.html`. Доп. настройка не нужна.

## Обновление картинок

Исходники — PNG в истории git; в репозитории хранятся оптимизированные `.webp`.
Чтобы добавить новую картинку, кладите сразу `.webp` в `public/images/` и
ссылайтесь на неё как `/images/имя.webp`. Исключения, оставленные в PNG:
`favicon_eagle.png` и `og_share.png` (для надёжного превью ссылок в Telegram).
