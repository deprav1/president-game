# Решение проблемы: Белый экран и 404 ошибки при деплое React+Vite на GitHub Pages

## Problem (Проблема)
При попытке развернуть и протестировать SPA-приложение (написанное на React + Vite) на хостинге **GitHub Pages** пользователь сталкивается с белым экраном. В консоли разработчика отображаются ошибки 404 (Not Found) для всех собранных ассетов (JS- и CSS-файлов).

## Root Cause (Причина)
Vite по умолчанию собирает проект с расчетом на то, что он будет развернут в корневом каталоге домена (`/`). В результате ссылки на собранные файлы в `dist/index.html` выглядят так: `/assets/index-xxx.js`.
Однако GitHub Pages публикует проекты по подпути вида `https://<username>.github.io/<repository-name>/` (в данном случае `https://deprav1.github.io/president-game/`). Браузер пытается загрузить файлы по адресу `https://deprav1.github.io/assets/index-xxx.js`, где их не существует, что приводит к ошибкам 404.

## Solution (Решение)
Решение состоит из двух шагов: настройки базового пути сборщика и настройки автоматизации сборки.

### 1. Настройка базового пути в Vite
В файле `vite.config.js` добавлен параметр `base` с указанием пути к репозиторию:
```javascript
export default defineConfig({
  plugins: [react()],
  base: '/president-game/', // Указывает Vite собирать проект с относительными путями
})
```
После этого пути в сборке стали генерироваться корректно: `/president-game/assets/index-xxx.js`.

### 2. Автоматический деплой через GitHub Actions
Создан файл автоматического сценария `.github/workflows/deploy.yml`, который реагирует на каждый пуш в ветку `main`. Он самостоятельно собирает проект и деплоит его на GitHub Pages без необходимости локального запуска сборки и ручного пуша в ветку `gh-pages`:
* Устанавливает Node.js и зависимости через `npm ci`.
* Собирает проект через `npm run build`.
* Архивирует и выкладывает папку `dist` на GitHub Pages.

С помощью GitHub CLI (`gh`) через API репозитория был включен источник GitHub Actions для страниц:
```powershell
gh api repos/deprav1/president-game/pages --method POST -f "build_type=workflow"
```

## Prevention (Предотвращение)
При создании и развертывании любого нового веб-приложения на React + Vite, которое будет хоститься на GitHub Pages:
1. Всегда прописывать `base: '/<repository-name>/'` в `vite.config.js` еще на этапе подготовки релиза.
2. Использовать шаблонный сценарий GitHub Actions для деплоя, что исключает человеческий фактор (забытые сборки, ручные ошибки при заливке).
