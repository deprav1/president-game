# Проблема с деплоем поддоменов и приоритетом index.htm на Timeweb

## Problem
При настройке GitHub Actions для деплоя дополнительного проекта (KidAI) на поддомен Timeweb (`kidai.cb077728.tw1.ru`) возникло две критические проблемы:
1. Сначала проект случайно перезатёр основной сайт (игру), так как был залит в корневую директорию.
2. После исправления пути деплоя на правильную папку поддомена, сайт по-прежнему не отображался, а вместо него висела стандартная заглушка Timeweb ("Сайт успешно создан"), несмотря на то, что логи GitHub Actions показывали успешную загрузку файлов.

## Root Cause
1. **Перезапись основного сайта:** В конфигурации деплоя `server-dir` изначально был указан как `./public_html/`. В архитектуре Timeweb это папка по умолчанию для *основного* домена. Папки поддоменов или дополнительных сайтов создаются рядом, например `./KidAI/public_html/`.
2. **Заглушка Timeweb:** При создании нового сайта панель управления Timeweb автоматически генерирует в папке файл-заглушку `index.htm`. Наш скрипт GitHub Actions загружал `index.html` проекта, но не удалял старые файлы. Веб-сервер хостинга настроен таким образом, что файл `index.htm` имеет приоритет над `index.html`. В результате сервер продолжал отдавать заглушку, игнорируя залитый код проекта.

## Solution
1. **Исправление пути:** В файле конфигурации `.github/workflows/deploy-timeweb.yml` параметр `server-dir` был изменён на правильную директорию поддомена: `./KidAI/public_html/`.
2. **Очистка директории (Clean Slate):** В шаг FTP-деплоя добавлен параметр `dangerous-clean-slate: true`. Эта настройка заставляет экшен полностью очищать целевую директорию перед заливкой новых файлов. Таким образом, дефолтный `index.htm` удаляется автоматически в процессе деплоя.

```yaml
      - name: Deploy to Timeweb FTP
        uses: SamKirkland/FTP-Deploy-Action@v4.3.5
        with:
          server: ${{ secrets.FTP_SERVER }}
          username: ${{ secrets.FTP_USERNAME }}
          password: ${{ secrets.FTP_PASSWORD }}
          local-dir: ./dist/
          server-dir: ./KidAI/public_html/
          dangerous-clean-slate: true # <- Ключевой параметр для удаления index.htm
```

## Prevention
- При настройке новых репозиториев для Timeweb **всегда** проверять реальное имя папки сайта через "Файловый менеджер" в панели управления.
- Использовать флаг `dangerous-clean-slate: true` для всех новых проектов, чтобы избежать конфликтов со сгенерированными хостингом файлами или старым кэшем.
- Помнить об иерархии приоритетов индексных файлов на хостингах (`index.htm` > `index.html` > `index.php`).
